# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project purpose

Classical-mechanics simulations written in Julia and rendered to MP4 for
use as teaching clips. Not a library, not interactive — every scene is a
standalone script that produces one video file.

## Commands

```bash
# First time only (slow — precompiles DifferentialEquations + CairoMakie)
julia --project=. -e 'using Pkg; Pkg.instantiate()'

# Render any scene (MP4 lands in output/videos/)
julia --project=. scripts/01_projectile.jl
julia --project=. scripts/02_pendulum.jl
julia --project=. scripts/03_double_pendulum.jl
julia --project=. scripts/04_spring_mass.jl
```

`ffmpeg` must be on `PATH` — CairoMakie's `record()` pipes into it to write MP4.

There is no test suite, lint, or build step. The "does it work" check is: run a
scene, then open the resulting MP4.

## Architecture

Two layers with a clear seam:

- **`src/mechanics/`** — pure physics. Each file defines an in-place ODE
  right-hand-side (`foo!(du, u, p, t)`) plus a `simulate_foo(...)` helper
  that builds an `ODEProblem` and calls `solve`, returning an `ODESolution`.
  No plotting, no I/O, no globals.
- **`scripts/NN_name.jl`** — one scene per file. Activates the project env,
  `include`s the relevant mechanics file, calls `simulate_*`, then drives a
  `CairoMakie.record()` loop over evenly-spaced sample times to produce an MP4.

Every script follows the same skeleton. When adding a scene, copy an
existing one — don't invent a new structure.

## Two conventions that matter

1. **Frame timing is decoupled from ODE steps.** Scripts always call
   `sol(t)` at evenly-spaced `t`s generated by `range(...; length=N)`, never
   at `sol.t` (the solver's adaptive steps). This is what keeps the animation
   framerate independent of problem stiffness. Preserve this when editing.

2. **Animations update `Observable`s inside the `record()` loop.** Build the
   `Figure` and plot objects once before `record`; inside the loop, mutate
   `Observable`s (`pt[] = ...`) or push into observable vectors and call
   `notify(...)`. Never rebuild the figure per frame.

## Adding a new scene

1. New physics model → `src/mechanics/<name>.jl` exposing `simulate_<name>`
   returning an `ODESolution`. Keep it pure — no plotting.
2. New script → copy `scripts/01_projectile.jl`, change the `include`, the
   `simulate_` call, the axis setup, and the `record` loop body.
3. Run it; the MP4 lands in `output/videos/<name>.mp4`.

## Backend note

Default is **CairoMakie** (headless, 2D, writes MP4 directly). GLMakie is
deliberately NOT in `Project.toml` so scripts render without a display
server. The Makie plotting API is backend-agnostic, so swapping
`using CairoMakie` for `using GLMakie` is the only change needed if an
interactive or 3D scene is later required.
